¿Por qué Observabilidad?
En producción, necesitas saber qué está pasando — antes que el usuario lo reporte
❓
Sin observabilidad
"El sistema está lento."

¿Cuál pod? ¿Cuál nodo? ¿Cuál servicio? ¿Desde cuándo? ¿Es código o infraestructura?

No sabes por dónde empezar.
📊
Los 3 pilares
Métricas: ¿Cómo se comporta el sistema? CPU, memoria, latencia

Logs: ¿Qué ocurrió exactamente? Eventos y errores

Trazas: ¿Cuánto tardó cada paso del request?
✅
Con observabilidad
"El servicio X tiene latencia p99 de 8s desde las 14:30. El log muestra un error de conexión a BD. La traza muestra el cuello de botella."

Problema resuelto en minutos.

Demo 2 — GitOps con ArgoCD
Un git push activa automáticamente el deploy en Kubernetes
1
Git Push
Modificamos k8s/deployment.yaml cambiando replicas: 2 → replicas: 3 y hacemos push a GitHub
2
ArgoCD detecta el cambio

Logs en Loki — Todo el Cluster Centralizado
Promtail recolecta logs de todos los pods automáticamente · Retención 7 días
🔦 Cómo funciona Promtail
  1. DaemonSet en cada nodo del cluster
  2. Lee /var/log/pods/*/*/*.log
  3. Parsea formato CRI (containerd)
  4. Extrae labels: namespace, pod, container
  5. Envía a Loki en tiempo real
🔍 Queries en Grafana Explore

# Errores en todo el cluster

# Logs de la demo app
{namespace="demo", pod=~"page-demo.*"}

Lecciones Aprendidas
Problemas reales encontrados y resueltos durante el bootcamp
  • 🔴→✅
    Grafana CrashLoopBackOff (subPath)
    No se puede montar con subPath dentro de un directorio ya montado. Solución: un solo ConfigMap con todos los dashboards.
  • 🔴→✅
    Tempo crash — mountPath /tempo
    /tempo ES el binario ejecutable. Solución: usar /var/tempo y emptyDir en vez de PVC NFS.
  • 🔴→✅
    Promtail sin logs ("no files to tail")
    kubernetes_sd con NODE_NAME regex no filtraba. Solución: static_configs con glob directo /var/log/pods/*/*/*.log
  • 🔴→✅
    Tekton PodSecurity violation
    Namespace en modo restricted bloqueaba buildkit. Solución: label pod-security.kubernetes.io/enforce=privileged
  • 🔴→✅
    Métricas Tekton con nombre incorrecto
    Tekton usa prefijo tekton_pipelines_controller_*, no tekton_*. Detectado via port-forward al pod distroless.
  • 🔴→✅
    GitHub bloqueó push con secrets
    Docker PAT en README.md detectado por secret scanning. Solución: borrar credencial, crear nueva, filtrar del historial.
